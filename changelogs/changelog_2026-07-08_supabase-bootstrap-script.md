# changelog_2026-07-08_supabase-bootstrap-script

## Зміна
Додано `supabase/bootstrap.sql` — єдиний ідемпотентний скрипт розгортання повної схеми БД у новому проєкті Supabase. Оновлено `CLAUDE.md` (розділ «Fresh Supabase project»).

## Як було до зміни
Попередній проєкт Supabase видалено, а схема була розкидана по 11 файлах `supabase/tables/*.sql` та 2 міграціях, які треба було виконувати вручну в правильному порядку (таблиці → RLS → платіжна міграція). Бакет `audio` взагалі створювався окремо через Dashboard або edge-функцію. Розгортання з нуля вимагало ~13 ручних кроків із ризиком помилитися в порядку залежностей.

## Що покращує зміна
- **Один крок замість тринадцяти:** вставити `bootstrap.sql` у SQL Editor і запустити. Скрипт створює всі таблиці (включно з legacy `suno_*` для відповідності репозиторію), функції `is_admin()`/`adjust_credits()`, усі RLS-політики, обидва storage-бакети (`audio` публічний, `merchant-docs` приватний) та seed планів і цін (UAH/USD/EUR).
- **Ідемпотентність:** повторний запуск безпечний (`IF NOT EXISTS`, `CREATE OR REPLACE`, `DROP POLICY IF EXISTS`, `ON CONFLICT DO NOTHING`) — скрипт можна використовувати і для «доливання» відсутніх обʼєктів у частково розгорнуту базу.
- **Перевірено на живому PostgreSQL 16:** скрипт прогнано двічі поспіль на локальному кластері зі стаб-схемами `auth`/`storage` (як у Supabase) — обидва прогони без помилок; перевірено кількість таблиць (11), політик (19+2 storage), бакети, seed (3 плани, 12 цін) та поведінку `adjust_credits` (списання, `insufficient_credits`, повернення, NULL-баланс).
