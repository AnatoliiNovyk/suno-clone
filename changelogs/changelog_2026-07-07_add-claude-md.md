# changelog_2026-07-07_add-claude-md

## Зміна
Додано файл `CLAUDE.md` у корінь проєкту — комплексна інструкція для AI-асистентів (Claude Code та інших).

## Як було до зміни
У репозиторії не існувало `CLAUDE.md`. Єдиною інструкцією для AI був `.github/copilot-instructions.md`, який містив застарілі технічні твердження (наприклад, що генерація музики — це "mock" в edge-функції `generate-music`, та що ключі Supabase захардкоджені в `lib/supabase.ts`). Актуальна архітектура — генерація через локальний Python-сервіс (Google Lyria RealTime) — ніде не була задокументована як основний шлях.

## Що покращує зміна
- `CLAUDE.md` описує **актуальний** стан кодової бази: фронтенд викликає локальний FastAPI-сервіс (`localhost:8000/generate-music`), а edge-функція `generate-music` позначена як legacy.
- Задокументовано робочі процеси для трьох компонентів (frontend `suno-clone/`, `python-service/`, `supabase/`), змінні середовища, конвенції стилів/типів та відомі обмеження.
- Явно зафіксовано робочі правила проєкту: відповіді українською, заборона змін без погодження (окрім `changelogs/`), обов'язковий changelog після кожної зміни.
- Зазначено, що `lib/supabase.ts` уже читає ключі зі змінних середовища (`VITE_*`), тобто застаріле твердження з copilot-інструкцій виправлено.
