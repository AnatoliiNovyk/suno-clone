# Changelog: Add Debug Logging to Python Service

## Дата зміни / Date of Change
2026-01-08

## Короткий опис / Short Description
add-debug-logging-to-python-service

## Опис змін / Description of Changes

### До зміни / Before Change
- Python-сервіс не виводив детальну інформацію про процес отримання даних від Google API.
- Було складно діагностувати причину "порожніх" аудіофайлів (чи це проблема з'єднання, відсутність відповіді від моделі, чи помилка збереження), оскільки в консолі був лише загальний статус.

### Після зміни / After Change
- Додано детальні `print` інструкції у файл `python-service/main.py`.
- Логуються події:
    - З'єднання з моделлю (`Connecting to Google GenAI model...`).
    - Отримання "ходу" моделі (`Model turn received`).
    - Розмір кожного отриманого чанка аудіо в байтах (`Received audio chunk: X bytes`).
    - Сигнал завершення генерації (`Turn complete signal received`).
    - Загальний обсяг отриманих даних.

### Переваги / Benefits
- Дозволяє точно визначити, на якому етапі втрачаються дані або виникає помилка при генерації аудіо.
- Пришвидшує налагодження проблеми з нульовою тривалістю треків.
